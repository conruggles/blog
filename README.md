# Connor's Blog

A personal blog built with [Blazor WebAssembly](https://learn.microsoft.com/aspnet/core/blazor/) and deployed as a static site to [GitHub Pages](https://pages.github.com/). No server required — everything runs in the browser.

## Tech Stack

| Layer | Choice |
|---|---|
| Framework | .NET 10 Blazor WebAssembly |
| UI components | [Blazor.Bootstrap](https://blazorbootstrap.com/) 3.x |
| CSS | Bootstrap 5 |
| Hosting | GitHub Pages |
| CI/CD | GitHub Actions |

## How It Works

Posts are plain HTML fragments stored under `wwwroot/content/YYYY/M/D/slug.html`. At build time, an MSBuild target in [blog.csproj](blog.csproj) scans all matching files and writes a `posts-manifest.txt` into the published output. At runtime, the Blazor app fetches that manifest and hydrates the archive navigation without any backend or database.

```
wwwroot/
  content/
    2026/
      5/
        9/
          my-first-post.html   ← post content fragment
    posts-manifest.txt         ← auto-generated by build
```

Routes follow the path of the file: `/2026/5/9/my-first-post.html`.

## Creating a New Post

Use the helper script — it creates the directory tree, generates a stub HTML file, and prints the route:

```bash
bash scripts/new-post.sh "My Post Title"
# or with an explicit date:
bash scripts/new-post.sh "My Post Title" 2026-05-09
```

No further registration is needed. Just commit the new file and push; the build picks it up automatically.

## Running Locally

```bash
dotnet run
# or for hot-reload:
dotnet watch run
```

The app is served at `https://localhost:5059` (or the port shown in terminal output).

## Deployment

Deployment runs automatically via GitHub Actions on every push to `main` (see [.github/workflows/deploy-github-pages.yml](.github/workflows/deploy-github-pages.yml)).

One-time repository setup required:

1. **Settings → Pages → Build and deployment source**: set to **GitHub Actions**
2. **Settings → Actions → General → Workflow permissions**: set to **Read and write permissions**

After that, every push to `main` builds and publishes the site. The workflow automatically sets the correct `<base href>` for both project repos (`/blog/`) and user/org repos (`/`).
